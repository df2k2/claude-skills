# Developer Fundamentals

Developer Fundamentals is the starting point in Ordergroove's developer docs. It walks engineers through the core concepts they'll need before integrating with the platform — the four-object data model (Customer, Subscription, Items, Order), the six components that make up the systems landscape, how authentication works with API keys and HMAC, and what the merchant launch process looks like. Think of it as your orientation before diving into a subscription commerce build.

***

## Data Model

Ordergroove’s data model is designed to provide flexibility to end users with a structure that allows for full automation or client-specific processes via rest APIs. Ordergroove’s data is stored in a Relational Database, with both one-to-one and one-to-many relationships depending on how you look at the data.

Ordergroove has four main data objects:

* Customer
* Subscription
* Items
* Order

For more information and a relational flowchart, take a look at [Data Model at a glance](https://developer.ordergroove.com/docs/data-model-at-a-glance).

***

## Systems Landscape Map

Ordergroove’s platform has six major integration components that power your relationship commerce experience. We have applications and cartridges for various eCommerce platforms that will automatically handle many of these components.

Even though the application automates some components, we still believe it is helpful to introduce those major components to help provide you with a full understanding of the integration.

Take a look at Ordergroove [Systems Landscape Map](https://developer.ordergroove.com/docs/landscape-map) for information on all six components and a relational chart.

## API Authentication

You can authenticate requests using either API key or an HMAC, depending on personal preference. API keys can be viewed and managed in the [Ordergroove Dashboard](https://ordergroove.zendesk.com/hc/en-us/articles/10601238595987); for more information take a look at [Authentication](https://developer.ordergroove.com/reference/authentication).
